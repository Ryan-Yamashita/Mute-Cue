# Security policy

## Supported versions

Only the newest public beta release is supported with security fixes.

## Reporting a vulnerability

Use GitHub's **Report a vulnerability** option in the repository Security tab when private vulnerability reporting is available.

If no private route is available, open a minimal public issue requesting private contact. Do not include proof-of-concept code, credentials, authorization data, logs, packet captures, or personal information in a public issue.

## Release trust

Mute Cue beta installers are currently unsigned. Download only from this repository's GitHub Releases page and verify the accompanying SHA-256 checksum. Signing status and any Windows warning must be stated clearly in each release.

The repository excludes local settings, encrypted authorization files, packet captures, signing keys, certificates, and generated release artifacts.
