# Security policy

## Supported versions

Only the current public beta release is supported.

## Reporting a vulnerability

Please do not open a public issue for a vulnerability that could expose credentials, authorization data, or another user’s system.

When this repository is published, use GitHub’s **Report a vulnerability** option in the repository’s Security tab. The repository owner must enable private vulnerability reporting in the GitHub settings before the first public release.

If private reporting is not yet available, do not publish sensitive technical details. Open a minimal public issue requesting a private contact route, without proof-of-concept code, logs, or personal data.
